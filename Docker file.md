# Docker file

Created: Nov 19, 2020 2:03 PM

Создает образ по заранее прописаному сценарию

Любой Docker file состоит из 4х инструкций. 

FROM - базовый или родительский образ, на основе которого будет создаваться наш образ

Workdir - задает рабочую директорию для  инструкции. 

COPY - копирует в контейнер файлы и папки. Так как мы формирует образ или контейнер. Подключить к нему папки и внешние тома не получиться. Используется функция копирования папок внутрь образа. 

Чтоб скопировать файлы нужно указать два пути

1. Путь до файлов на хост машине относительно docker file
2. Путь куда положить файлы внутри контейнера

CMD - описывает стартовую команду с аргументами которые нужно выполнить когда контейнер будет запущен. 

В одном Docker file Может присутствовать одна инструкция CMD. 

Docker build - для того, чтоб собрать образ на основе Docker file - собирает образ. Контейнер нужно запускать самостоятельно.

-t позволяет задать имя образа.  В данном случае python_counter
